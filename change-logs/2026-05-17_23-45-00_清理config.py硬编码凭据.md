1. **🔴 立即在 PostgreSQL 中轮换 `poker_user` 密码**——这是 R-2 合规 step 4 的强制要求：

   ```sql

   ALTER USER poker_user WITH PASSWORD '<new-strong-password>';
   ```

   **理由**：
   - git history `5670ef8 初始版本` commit 永久包含明文密码 `Lc%40135681`
   - 即使本次 commit 把代码改了，旧密码仍在 history 中；任何 clone 过本仓的人 + GitHub remote + 备份系统都仍可读到

2. **在新 `.env` 中填入新密码**：

   ```bash

   POKEMIR_DB_DSN=postgresql+asyncpg://poker_user:<new-strong-password>@localhost:5432/poker_assistant
   POKEMIR_DB_DSN_SYNC=postgresql://poker_user:<new-strong-password>@localhost:5432/poker_assistant
   ```

3. **新密码记录到密码管理器**（1Password / Bitwarden / KeePass 类）——不要再硬编码
