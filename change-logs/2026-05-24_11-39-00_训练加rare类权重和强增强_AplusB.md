## 1. 任务概述

- **用户反馈**：上轮训练 val_both_acc 94.87%, fixture pytest 4 fail：
  - `4h_pm_001` (4h→7h) — 4h 只有 1 张训练样本
  - `Jh_pm_002` / `Jh_pm_004` (Jh→Js) — Jh 5 张但 3 张误判,模型对 Jh/Js 视觉边界不稳
  - `5h_pm_002` — 已在上一 commit (530fe96) 修标签
- **用户决定**：4h / Jh "可遇不可求", 不靠数据靠**算法 + 训练侧补救**
- **本次实现 A+B**：
  - **A**：rare 类样本在 loss function 中获得**额外权重**（默认 5×）
  - **B**：rare 类样本在 augmentation 中使用**更强的扰动**（亮度 ±50% / 旋转 ±10° / 增加平移）

## 2. 假设清单

| # | 假设 | 出处 |
|---|---|---|
| 1 | "rare" 定义 = rank-suit 组合 ≤ 2 张 base fixture | 上轮 1-张类全失败,2-张类（如 5h）也部分失败,3+ 类稳 |
| 2 | rare loss weight = 5× 是合理起点 | 文献常用 3-10× 范围;太低无效,太高过拟合;5 居中 |
| 3 | strong augmentation 对 rare 而 normal 对常类是合理分割 | 常类已经数据多,过强增强反而稀释信号 |
| 4 | 用 `CrossEntropyLoss(reduction='none')` + per-sample 乘法实现 weight | 比 `weight=` 参数更灵活（后者是 per-class,我们要 per-sample）|
| 5 | rare 集合按全部数据计（不分 train/val）| 简单一致;split 后某类全在 val 不在 train,影响有限（model anyway 学不到）|
| 6 | 评估时新增 rare-class subset 的 both-acc 跟踪 | 帮看"rare 是否真的学进去了"——之前没有这个 visibility |

## 3. 文件变更清单

| 文件路径 | 变更类型 | 变更说明 |
|---|---|---|
| `tools/train_card_cnn.py` | 修改 ~70 行 | (a) `CardDataset`：检测 `rare_set` (samples 计数 ≤ `RARE_THRESHOLD=2`)；分 `tf_normal` / `tf_strong` / `tf_val` 三套 transform；`__getitem__` 返回 4-tuple 含 `is_rare` flag。(b) `evaluate()`：返回额外 `rare_acc` + `total_rare`（对 rare 子集单独算 both-acc）。(c) 主循环：`CrossEntropyLoss(reduction='none')` + per-sample weight = `1 + (rare_weight - 1) * is_rare`。(d) 加 CLI `--rare-weight` 默认 5.0。(e) 启动横幅打印 rare 类清单（train/val 各自）+ rare weight。(f) 每 epoch print 末尾加 `rare=XX%(N)` |

## 8. 潜在影响范围

- **正向**：
  - rare 类（4h / Jh / 9s / Ks / Ac 等）训练时获得 5× 损失关注 → 模型更努力学
  - rare 类 augmentation 更强（亮度 ±50% / 旋转 ±10° / 平移 ±5%）→ 50 张 augmented 多样性提高
  - epoch 日志新增 rare-class 跟踪,**可视化 rare 学习进度**
  - val rare-acc 单独跟踪,作为辅助决策信号（如果某 rare 类持续 0% 说明需补真实样本）
- **行为变化**：
  - Dataset `__getitem__` 改成 4-tuple — 仅本文件用,无下游 break
  - 训练时间几乎不变（augmentation 强度略增,可忽略 < 5%）
  - `CnnClassifier`（推理端）完全兼容；新旧模型权重都能加载
- **预期 val 提升**：94.87% → **96-98%**（基于经验性 rare-class 改进幅度）
- **关联待办**：
  - 用户重训 + 报数字
  - 如果 rare-class subset 仍 < 70% 准 → 加大 rare-weight 到 7-10 或补真实数据
  - 如果 val total 没明显升 → rare 类不是主要瓶颈,可能要看 epoch 数或 augment 参数
