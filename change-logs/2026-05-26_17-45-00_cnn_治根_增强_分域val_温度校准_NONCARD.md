## 1. 任务概述

诊断结论(上轮):**CNN 对摊牌牌图 rank 普遍 conf 0.45-0.89,远低于 0.9 阈值**。根因是训练数据只含 community(干净/正面),没见过 showdown(略小/带角/JPEG 压缩重)。

**治根方案 4 件套**(本 commit 全部完成):

| # | 内容 | 价值 |
|:---:|:---|:---|
| #3 | 数据增强:JPEG noise + Perspective + 现有 rotation/blur/jitter 复用 | 缩小训练/真实分布 gap |
| #4 | 多域 fixture 读取 + 分域 val 追踪 + 谐波均值最佳检查点 | 防止"加新数据反而旧域退化" |
| #5 | 温度校准(L-BFGS 学 T)+ ckpt 存 T + 推理 softmax(z/T) | conf=0.9 真等于 90% 准 |
| #6 | iscard 二分类头(NONCARD 反幻觉)+ 推理时 P(CARD)<0.5 直接 return None | 治 seat_X 头像稳定幻觉 |

**关键设计**:用户 0 录摊牌时**所有改动都向后兼容**(只跑 community,iscard 头关闭,T=1.0);用户**录满后训练时**自动开 3 头 + 校准 — **零迁移成本**。

## 6. 风险

- ⚠️ iscard 头**没 NONCARD 样本时随机预测** — 但 `_has_iscard` 守卫确保只有"训了 NONCARD"的 ckpt 才开启 gate
- ⚠️ 温度校准依赖 val 集有正确分布的样本 — 当 showdown val=0 时,只校 community 域
- ⚠️ Perspective + JPEG noise 增加了训练计算量(~10-15% 慢) — 可接受
- ⚠️ batch 中可能全是 noncard(虽然概率低)→ `n_cards.clamp(min=1.0)` 防除零
