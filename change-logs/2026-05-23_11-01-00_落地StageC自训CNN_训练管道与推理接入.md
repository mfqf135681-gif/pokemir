## 1. 任务概述

- **用户决议**：Stage C（自训 CNN）over Stage A（SmolVLM）over heuristic，理由 = 延迟敏感（CNN <50ms vs VLM 200-500ms）+ 5070 Ti 算力充裕 + 愿意投入训练劳动换长期产物
- **落地范围**：写训练管道 + 推理类 + 接入 cards.py 三级 fallback；**模型权重不在本 commit**（用户 Win 端跑训练产出 `models/card_cnn.pth`，该路径 gitignored）
- **scope 不做**：不在 Linux 端运行真正训练（仅 smoke 测）；不修 requirements.txt（torchvision 依赖让用户按需装，避免触发自动 commit 黑名单）

## 2. 假设清单

| # | 假设 | 出处 |
|---|---|---|
| 1 | 双头架构（rank head + suit head 分头）优于 52-way 单分类 | 31 fixture / 52 classes ≈ 0.6 样本/类 → 不可学；改成 13+4 头，每类样本 2.4 + 7.8 倍提升 |
| 2 | ~256K 参数模型够用且不过拟合 | 任务复杂度低（图像区分度高），数据量限制下小模型更稳 |
| 3 | 强增强（亮度 ±30% / 旋转 ±5° / crop jitter / HSV / 高斯模糊）覆盖暗态卡 + ROI 噪声 | [[wepoker-card-brightness-variance]] 要求亮度变体；ROI 配偶尔不准 |
| 4 | 每 base fixture × 50 augmentation = ~1500 训练样本，对小 CNN 充足 | 经验：小 CNN 在 1000+ 样本上往往 90%+ 准确率 |
| 5 | 模型存 `models/card_cnn.pth`（gitignored），用户 Win 训完不进 git | 模型可重训；体积小但本质是衍生数据；保持 `models/` 全 gitignore 干净 |
| 6 | CnnClassifier 模型缺席时静默 fallback（return None）| CardRecognizer 三级链已设计：CNN → vision → heuristic |
| 7 | Linux smoke 测仅验脚本逻辑跑通（2 epoch CPU），不验证准确率 | [[compute-architecture-asymmetry]] —— VPS 弱算力；真实训练 + 评估由 Win 端用户做 |

## 7. 手动操作提醒

⚠️ **Win 用户实测流程**（合并 Stage C 训练 + 评估）：

1. `git pull`
2. **装 torchvision**（如未装）+ 确认 GPU 可用：
   ```powershell
   .\.venv\Scripts\python.exe -m pip install --upgrade --index-url https://download.pytorch.org/whl/cu124 torch torchvision
   ```
   验证：
   ```powershell
   .\.venv\Scripts\python.exe -c "import torch; print(torch.cuda.is_available(), torch.cuda.get_device_name(0) if torch.cuda.is_available() else '')"
   ```
   应输出 `True RTX 5070 Ti` 或类似
3. **跑训练**：
   ```powershell
   .\.venv\Scripts\python.exe tools\train_card_cnn.py
   ```
   预期：60 epoch；GPU 加速下几分钟；自动保存 best-val 到 `models\card_cnn.pth`
4. **跑诊断 + baseline**：
   ```powershell
   .\.venv\Scripts\python.exe tools\diagnose_recognition.py
   .\.venv\Scripts\python.exe -m pytest tests/test_recognition_fixtures.py -v --tb=no
   ```
5. 把训练末尾的 best-val 数字 + diagnostic summary + pytest summary 发我

⚠️ **后续 fixture 增量时**：每加一批新 fixture 后需重跑步骤 3（重训）；步骤 4 看新 baseline。
