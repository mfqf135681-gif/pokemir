1. `pipeline/io/diag_queue.py` — DiagQueue 异步 diag 写入(替代 38 处 inline emit)

   - **Block-on-full** 策略(DB 写入顺序敏感:hand start → action events → hand end 不能丢)
3. `tests/test_io_queues.py` — 14 单元测试(mock writer + mock session,无真 DB 依赖)
