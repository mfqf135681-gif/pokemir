- **关联需求**:Win 端 ATTENTION_MODE=1 实测 verify(T93)暴露 bug
- **触发红线**:无

## 1. 任务概述

T93 Win 端 1 小时 × 2 次实测发现 ATTENTION_MODE=1 灾难性 regression:
- session_1(mode=0,21 hands):silent fold 25.6% ✅ 健康
- session_2(mode=1,76 hands):silent fold **70.2%** 🔴 灾难
- 第 2 次 44 hands 实测:silent **100%**(0 fold captured)
- Tick median 45ms = OCR 基本没跑

**Root cause(95% 信心)**:
`SeatLifecycle._SKIPPABLE` 含 `EMPTY` → pipeline boot 时全 seat EMPTY → 全 skippable → pre_batch action OCR 全跳过 → 没 fold 事件 → mirror 不触发 → 状态永停 EMPTY → 死循环

**Fix**:1 行 enum 调整 + 测试 update

## 2. 假设清单

1. EMPTY 语义重定义:**"未知 / 未初始化"** 而非 "已确认空座"
2. 已确认的 sit-out 由 `_detect_empty_seats` 触发 mirror → SITTING_OUT(才跳过)
3. EMPTY 不 skip 跟 legacy mode `_empty_seats` 初始空集 "默认不跳" 行为等价
