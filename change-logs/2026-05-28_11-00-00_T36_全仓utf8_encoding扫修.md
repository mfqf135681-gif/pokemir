# T36:全仓 `open()` / `read_text()` 加 `encoding="utf-8"`

- **完成时间**:2026-05-28 11:00 UTC(北京 19:00)
- **触发**:Win 端跑 `python tools\roi_config.py --verify` 炸 `UnicodeDecodeError: 'gbk' codec can't decode byte 0xa5 in position 108` — `party_poker_9.json` 含中文 window_title
- **关联前次**:T23 同款 bug 修过 meta.json,但**只修了 1 处,没扫全仓**
- **触发红线**:无
- **无关红线已检查**:R-1 ~ R-10

## 1. 任务概述

Win Python 3.14 默认 cp936/gbk(中国区 locale),`open(path)` / `Path.read_text()` 不显式 `encoding=` 时,**含中文的 JSON / .env / md 读取即炸**。Linux 默认 UTF-8 **永远不触发**,VPS 沉默 ship。

本次扫修全仓,**15 处**(初始 grep 16 处含 1 个 false positive)。

## 2. 文件变更清单

| 文件 | 修复点数 | 说明 |
|:---|:---:|:---|
| `tools/roi_config.py` | 9 | 4 处 read + 5 处 write,JSON profile 含中文 window_title |
| `pipeline/orchestrator.py` | 3 | 读 PLAYER_REGISTRY_PATH + 读 roi_path(2 read, 1 write)|
| `capture/roi.py` | 2 | ROIManager.from_json + to_json |
| `tools/replay_review.py` | 1 | `env_path.read_text()` 加 encoding |
| **tools/record_card.py** | **0** | **初 grep false positive**,已显式 `encoding="utf-8"` |

**总计:15 处 fix**

### 同时改进:write 路径加 `ensure_ascii=False`

5 处 `json.dump(...)` 同步加 `ensure_ascii=False`,**中文字面落盘,不再 \u 转义**,文件更可读 + 体积更小。向后兼容(JSON 标准要求支持 UTF-8)。

## 3. 假设清单

1. Linux Python 3.12 / Win Python 3.14 均支持 `encoding="utf-8"` 参数 — 自 Python 3.0 起就有
2. JSON 文件内容是合法 UTF-8 — 历史数据已 dump 验证(VPS Linux 写 = UTF-8)
3. `ensure_ascii=False` 不破坏 downstream parser — JSON 标准 mandates UTF-8
