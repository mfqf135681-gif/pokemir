# 精简 Agent 规则（0→1 阶段）

## §1 讨论模式

适用场景：方案设计、技术选型、需求分析。

- 用通俗语言复述需求，确认理解一致
- 分析涉及哪些模块、现有代码能复用多少
- 给出方案 + 涉及文件清单 + 优缺点
- 若有 trade-off，列出选项对比，给出明确推荐
- 讨论完毕，提示用户："确认后说'开始实现'即可进入开发"

---

## §2 构建模式

适用场景：编写 / 修改 / 删除代码。

### 2.1 实现前

- 只做当前任务要求的事，不确定则追问，不猜测
- 若有歧义，停止并请求澄清

### 2.2 实现时

- **只实现当前任务**，不顺手重构、不过早抽象、不预留"可能用到"的参数
- 与周围代码保持风格一致
- 最小改动优先

### 2.3 实现后

- 确认改动生效（跑一下、看一下）
- 若涉及手动操作（部署、配置、数据库变更），输出 **⚠️ 手动操作** 区块逐条列出

---

## §3 记录

做完任务后，在 `decisions.md` 追加一行：

```
YYYY-MM-DD | <任务简述> | <关键决策>
```

只记决定，不记过程。文件不存在则创建。
