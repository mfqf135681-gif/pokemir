# AGENTS 路由器

## 意图分类

在回答用户任何请求之前，先判断意图：

1. **用户是否在讨论方案、设计、需求，而不要求产出代码？**
   - 关键词：`讨论`、`设计`、`方案`、`怎么实现`、`技术选型`、`建议`
   - → **DISCUSSION** → 读取 `.agents/rules.md` §1

2. **用户是否要求写代码、修改代码？**
   - 关键词：`实现`、`写`、`改`、`修复`、`新增`、`重构`
   - → **BUILD** → 读取 `.agents/rules.md` §2

3. **用户是否要求运行、测试、调试、排查问题？**
   - → **BUILD**（0→1 阶段，测试并入构建流程）

4. **模糊输入**（如"看看这个"、"这个功能"）→ 追问意图，不猜测归类。

复合指令（如"分析然后实现"）→ 先走第一步对应的模式。

## 激活信号检测

每次对话结束前，检查当前项目状态：

| # | 信号 |
|:--|:---|
| 1 | 项目文件数 > 40 |
| 2 | 存在 4+ 个独立功能模块 |
| 3 | 用户频繁提出跨模块影响类问题 |
| 4 | 项目中已存在 `contracts/` 或 `docs/` 目录 |
| 5 | 单次请求涉及 5+ 个文件修改 |

**若满足 3 条以上**，在回复末尾输出：

> 当前项目复杂度已达到一定规模，是否考虑激活完整治理体系？归档在 `.agents-full/`，详见其中的 README。

若未触发，不输出此提醒。
