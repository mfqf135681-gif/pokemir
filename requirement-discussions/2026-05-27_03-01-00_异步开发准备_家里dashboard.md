## 2. 双机分离架构图

```
[Win 桌面 — 当前位置]            [家里电脑 — 后续]
  • WePoker H5                      • Streamlit dashboard
  • Pokemir pipeline                • IDE / git client
    (capture/recognition/CNN)

       │                                   │
       │                                   │
       └──────── Tailscale mesh ───────────┘
                  (R-3 合规)
                       │
                       ▼
         ┌────────────────────────────┐
         │   VPS PostgreSQL 15        │
         │   (Linux,Tailscale 节点)   │

         └────────────────────────────┘
```

### 2.1 数据流方向

| 方向 | 内容 | 频率 |
|:---|:---|:---|
| Win → DB | hands / action_events / diagnostic_events INSERT | 实时(每 250ms tick)|
| 家里 → DB | SELECT only | 用户触发(按页刷新)|
| Win ←→ 家里 | git push/pull | 工作日中数次 |

### 2.2 关键约束
- **DB 是单一可信源**(SoT):**绝不允许两端各自存 SQLite 类的本地副本**
- **schema 改动必须先 push 到 git**,**两端 pull 后才生效**
- **Tailscale ACL** 控制谁能连 DB(目前只你的设备)

- `requirements.txt` — 全套 deps(pipeline + dashboard)
- `requirements-dashboard.txt` — 仅 dashboard 子集(streamlit + sqlalchemy + 必需)

**默认采纳 B 方案**:让家里只装 dashboard 子集,**装机时间从 20 min 缩到 2 min**。

### 3.5 边界 + 兜底

| 边界场景 | 处理 |
|:---|:---|
| 家里 Python 版本 < 3.10 | dashboard 用了 type unions `int \| None` 需 3.10+;**setup 文档要求 3.12+** |

| Streamlit 与现有依赖冲突 | 在 venv 隔离;**禁止全局 pip install** |

### 4.4 db.py 连接封装

```python
import os
from sqlalchemy import create_engine, text
from sqlalchemy.exc import OperationalError

_engine = None

def get_engine():
    global _engine
    if _engine is None:
        dsn = os.getenv("POKEMIR_DB_DSN_SYNC",
                        "postgresql://poker_user:poker_pass@100.101.105.46:5432/poker_assistant")
        _engine = create_engine(dsn, pool_pre_ping=True, pool_size=2)
    return _engine

def db_health_check() -> tuple[bool, str]:
    """快速 DB 健康 check,30s 超时."""
    try:
        with get_engine().connect() as c:
            n = c.execute(text("SELECT COUNT(*) FROM hands")).scalar()
        return True, f"{n} hands"
    except OperationalError as e:
        return False, f"connect fail: {str(e)[:50]}"
    except Exception as e:
        return False, f"err: {str(e)[:50]}"
```

### 4.5 边界 + 兜底

| 场景 | 处理 |
|:---|:---|
| DB 连接失败(Tailscale 断 / VPS 离线) | sidebar 显 🔴 + 错误信息;**dashboard 仍能启动,只是 query 返空** |
| 查询超时(慢网络) | 加 `pool_pre_ping=True`,每次查询前 ping;**设 30s timeout** |
| view 未实施(还没建表) | stats.py 函数捕获 `UndefinedTable` exception,返回 `None` → 上层显示"等待 view" |
| 数据量太大(假设 hands > 10000) | 用 `st.cache_data(ttl=60)` 缓存 60s |
| 中文显示乱码 | Streamlit 默认 UTF-8 但确保 `st.set_page_config(..., page_icon="🃏")` 不挂;字体本地化 |
| Streamlit 配置丢失 | 配 `.streamlit/config.toml` 进 git,**两端一致** |

### 6.3 边界 + 兜底

## 7. 跨组件横切关注点

### 7.1 网络 / Tailscale

- VPS PG 端口 5432 仅 Tailnet 可达,**公网不开放**(R-3)
- 家里电脑 Tailscale 离线 → dashboard 显 🔴 + 引导用户 `tailscale status`
- 网络抖动 → DB 连接池自动重连(`pool_pre_ping=True`)

### 7.2 Git 协作

- main 分支:**两端都 push**,小项目无 PR 流程
- 冲突:绝大多数 frontend / backend 文件不重合,自然解决
- 建议:写 backend 之前先 `git pull`;写 frontend 之前先 `git pull`

### 7.3 版本一致性

- `requirements-dashboard.txt` 加 `streamlit==X.Y` **不要 ==,用 >=Y,<Y+1** 避免锁死太严
- 主要 lib 跨大版本时**两端协调**(git commit message 提示)

### 7.4 调试支持

- dashboard 出错时 Streamlit 自带 stack trace 显示
- DB 慢查询用 `EXPLAIN ANALYZE` 调
- 远程问题:截图 + 日志贴 / 这边 SQL 查 → 异步排查
