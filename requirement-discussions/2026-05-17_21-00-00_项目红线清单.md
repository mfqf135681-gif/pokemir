## 阶段 3：候选红线池（8 条）

### R-1：禁止任何形式的牌室客户端交互/自动化
- **描述**：项目不得集成键鼠注入、进程内存读取、网络抓包、API 调用等任何主动与 poker 客户端交互的行为。仅允许只读、被动的窗口位置查询 + 屏幕图像捕获
- **触发条件**：
  - 调用 `pyautogui` / `keyboard` / `pynput` / `pymem` 等键鼠/内存 API
  - 抓包 poker 流量（`scapy` / 中间人代理）
  - 调用除 `EnumWindows` / `GetWindowRect` / `GetWindowText` 之外的 Windows hook API
- **合规动作**：立即停止；任何讨论必须先 REQ 模式评估 ToS 风险
- **不入风险**：项目某天被顺手加自动点击 → 账号封禁 + 法律纠纷

### R-2：禁止源码硬编码数据库或第三方凭据
- **描述**：所有源码文件不得包含真实密码、token、API key；敏感值走 `.env` + `os.getenv` 占位符 default
- **触发条件**：
  - `os.getenv` 的 default 值含非占位符密码（如 `Lc%40135681`）
  - `requirements.txt` / `.env` 之外文件出现 `password=` / `token=` / `api_key=` 字面量
- **合规动作**：迁移到 `.env`；源码 default 改 `change-me` 类占位符；git history 中泄露的凭据立即轮换
- **不入风险**：git history 永久泄露密码；公开仓后 DB 被扫库洗

### R-3：禁止未经 REQ 确认修改 contracts/
- **描述**：`contracts/api.yaml`、`contracts/models.sql` 等契约文件的修改必须有对应 `confirmed`/`accepted` 状态的 `requirement-discussions/` 文档授权
- **触发条件**：DEV 模式下编辑 `contracts/` 任何文件
- **合规动作**：先在 REQ 模式完整讨论、保存为 `confirmed` 记录；change-log §4 显式 cross-link
- **不入风险**：治理体系名存实亡——契约形同虚设

### R-4：禁止将本地手牌数据外传
- **描述**：数据库中的 `hands` / `action_events` / `player_stats_cache` / `player_situational_stats` / `replay_corrections` 是用户私有数据。代码不得集成主动上报、第三方分析、云同步行为。纯监控类（PostHog/Sentry 不读业务数据）不在此限
- **触发条件**：
  - 代码中 `requests.post` / `httpx.post` 等向非 localhost / 用户显式配置内网地址发送上述表数据
  - 集成读取业务数据的第三方 SDK
- **合规动作**：所有数据保留本地；确需上报必须 REQ 讨论 + 数据脱敏 + 用户显式同意
- **不入风险**：隐私事故；开源后第三方贡献者顺手加分析 SDK

### R-5：屏幕捕获范围限定
- **描述**：`capture/screen.py` 的全屏抓取能力（`capture()` / `capture_raw()`）只能用于 ROI 配置工具（`tools/roi_config.py`）；运行时 pipeline 必须走 `capture_roi()` + 已声明的 window
- **触发条件**：
  - `pipeline/` 模块调用 `capture()` 或 `capture_raw()`
  - `tools/` 之外的模块调用 `capture_raw()`
- **合规动作**：改为 `capture_roi()` + 显式 ROI；或 REQ 讨论新增 ROI 类型
- **不入风险**：pipeline 某天为省事全屏抓 → 抓到无关窗口跑 OCR → 隐私事故

### R-6：Win 测试副本只读（建议放 docs，不立红线）
- **描述**：rsync 同步的 Win 端 `pokemir-test/` 禁止编辑，所有修改在 Linux dev 端
- **不入红线理由**：这是工作流约定，不是代码可机器化检测的红线。归 `docs/dev-workflow.md` 更合适
- **建议**：写入 `docs/dev-workflow.md`，不占红线槽位

### R-7：Windows API 调用范围隔离
- **描述**：`ctypes.windll` 及其下属调用只能存在于 `capture/screen.py` 和 `tools/roi_config.py`；其他模块禁止直接调用 Windows API
- **触发条件**：`pipeline/` / `recognition/` / `events/` / `storage/` / `stats/` / `hud/` / `api/` 任一模块 `import ctypes` 或 `from ctypes import windll`
- **合规动作**：将平台特定逻辑下沉到 `capture/`；通过抽象接口调用
- **不入风险**：`pipeline/` 慢慢加 windll → Linux 上完全跑不通 → 测试覆盖率塌方

### R-8：识别精度回归保护（待激活）
- **描述**：识别模块（`recognition/*`）的修改不得让 `tests/fixtures/` 的整体识别率下降超过 2%
- **触发条件**：`pytest tests/test_recognition_fixtures.py` 输出识别率较 main 下降 >2%
- **合规动作**：修复使其不下降；或 REQ 讨论例外（算法换代换得更多收益）
- **激活前提**：fixture 库建立（Task #2 完成后）
- **建议**：本次先不入，Task #2 完成后回头补立
