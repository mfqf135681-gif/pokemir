### 1.1 核心架构

```
┌──────────────────────────────────────────────────────────┐
│  Pokemir MVP                                              │
│                                                           │
│  WePoker H5 (用户正常打)                                │
│       ↓ 屏幕像素                                          │
│  Pokemir Pipeline (后台)                                 │
│       │  - mss 抓帧 (4Hz)                                 │
│       │  - EasyOCR + CNN 识别                             │
│       │  - 写 PostgreSQL                                  │
│       ▼                                                   │
│  PostgreSQL                                               │
│       │  - hands / action_events / diagnostic_events     │
│       │  - 6 SQL views (守恒 / timing / insurance / ...)  │
│       ▼                                                   │
│  Streamlit Dashboard (浏览器,可 Chrome PWA 装桌面)         │
│       │  侧栏 4 模块:                                     │
│       │   ① 复盘 (玩家画像 / Hand 历史 / 图表)             │
│       │   ② 实时 (当前桌动态画像)                          │

│       ↓                                                   │

└──────────────────────────────────────────────────────────┘
```

### 1.2 模块功能定义

| 模块 | 功能 | 触发 |
|:---|:---|:---|
| ① 复盘 | 对手列表 + 个体画像 (VPIP/PFR/AF/3-bet/snap%/insurance) + Hand 历史回放 + plotly 图表 | 用户主动浏览 |

### 1.3 HUD 取消理由

**用户澄清(2026-05-27)**:
> "我不要求浮在游戏上层,游览器和这个应用应该分别是两个独立的应用。"

原 Path D HUD overlay(浮在 WePoker 上 / 透明 always-on-top 窗口)路径**完全取消 MVP**:
- 用户预期是"游戏窗口 + Pokemir 窗口分屏",而非"游戏窗口 + 透明 HUD 浮层"
- 取消 HUD 后,**Streamlit / 浏览器 PWA 重新可选**(原本因 HUD 必须才被排除)
- **解锁简化路径** — 不需要 Win32 透明窗口 API / 不需要 PySide6 桌面框架

**HUD 不删除,降级到 #LR5 远期**:若未来用户想要"实时辅助"再启用,但需重新评估 ToS 风险。

### 1.4 双机分离支持

用户指出:
> "我可以完全分离两台计算机,用一台计算机采集另外一台计算机的视频信号→读取→分析→反馈。"

**架构允许双机部署**,但不强求:

```
[机器 A:采集机]                     [机器 B:分析机]
  - WePoker 客户端                    - Streamlit dashboard
  - 轻量 capture daemon               - Pipeline + PG + CNN + LLM
  - 截图每 250ms 上传                     │
       │                                     │
       └────── Tailscale 内网 ──────────────►
            (R-3 合规,已有)
```

**优势**:
- A 极简(只需 mss + 网络),老笔记本可用
- B 全栈 + GPU 任务(5070 Ti)
- WePoker 看不见分析逻辑,**R-1 / R-3 双红线全过**
- A 死机不影响数据(全在 B)
- B 可以 Linux(R-3 跨平台部署)

**MVP 实施**:单机优先(简单),双机模式留 hook(capture daemon 可独立模块)。

## 2. LLM 解释层红线

### 2.1 红线全文

> **在 pokemir 项目中,LLM(本地 Qwen / 远程 Claude 等任何 LLM)只能用于"解释层"任务:**
>
> - ✅ 牌局自然语言解释("这手你为什么应该 raise 而不是 call")
> - ✅ 策略建议("对手画像 = nit,你 fold 这个 range 是对的")
> - ✅ 用户主动提问的对话式查询("我今天 VPIP 高在哪些位置?")
> - ✅ 任何需要"自然语言推理 + 用户在 loop 内审查"的任务
>
> **绝对禁止**:
>
> - ❌ DB janitor / 自动数据修复 / ID dedup
> - ❌ 定时任务 + 无人审查的自动写库
> - ❌ 规则化任务(替代死规则代码)
> - ❌ 任何 LLM 错了会"静默错位 + 不可逆 + 影响统计"的位置

### 2.2 红线提出背景

用户 2026-05-27 提议:"本地 LLM + Postgres MCP + 10 min 定时任务,自动修 OCR 飘字之类的 ID dedup / 数据质量问题"。

评估为**运维层任务**,**红线触发**,**否决**。

**否决理由**:
1. **现有方案已覆盖 80%**:`_canonicalize_player_id_map` 模糊匹配 + avatar phash fingerprints + DB UPDATE 同步
2. **风险与收益严重不对称**:误合并真玩家 → 所有统计错位 + 不可逆;LLM 监控时间 > LLM 节省时间
3. **5070 Ti 资源冲突**:Qwen2.5-7B 量化 5-6GB 常驻 + pipeline + 偶尔 retrain → 显存吃紧
4. **死规则 + 人工 review 队列**用 1/10 成本做 LLM 能做的 80%,可验证 / 可回滚

**列入** `[[long-term-roadmap]] #LR9`(被红线否决路径,留作未来证据反驳用)。

### 2.3 路由判别(下次同类提议)

第 1 问:**"这是解释层还是运维层任务?"**
- 解释层(自然语言 I/O,用户审查)→ 走 #LR7-A,**可考虑**
- 运维层(自动写库 / 定时任务 / 数据修复 / 规则化决策)→ **红线触发**,**否决**

第 2 问:**"等价的死规则方案多复杂?"**
- < 100 行 Python → **死规则赢**,LLM 提议否决
- 真需要自然语言推理(罕见)→ 才考虑解释层 LLM

## 3. Win 打包路径(分阶段)

### 3.1 三阶段方案

| 阶段 | 时机 | 方案 | AV 风险 |
|:---|:---|:---|:---|
| **现状(MVP)** | 自己 + 1-2 朋友 | Python 源码 + `start.bat` 启动 | 🟢 零 |
| **#LR(后期)** | 准备小圈分享(< 10 人) | Inno Setup 安装器(打包 Python + venv + 代码) | 🟢 极低 |
| **不实施** | 任何时候 | PyInstaller 打包单 exe | 🔴 高(Defender 误报)|
| **永远不做** | — | 公开 / 商业化 | 🔴 ToS 风险高,违反 [[wepoker-tos-risk-boundary]] |

### 3.2 永远不用 PyInstaller 的理由

1. Windows Defender 经常误报 `Generic.PyInstaller` / `Trojan.Generic`
2. PyInstaller + UPX 压缩双重触发,几乎 100% 误报
3. 项目 ~2GB(PyTorch 等),单 exe 性能不佳
4. 不需要"双击 1 个 exe 启动"的极简 UX,**用户都能跑 Python**

### 3.3 替代:Inno Setup 安装器

```
PokemirInstaller.exe 内部:
  1. 解压 portable Python 3.12 → C:\Program Files\Pokemir\python\
  2. 解压项目代码 → C:\Program Files\Pokemir\app\
  3. pip install -r requirements.txt(进 venv)
  4. 创建开始菜单快捷方式
  5. 注册 uninstaller
```

**用户体验跟原生 exe 一样,但 0 AV 误报**(Python.exe 是 Microsoft 信任的可执行文件)。

### 3.4 远期商业化方案(后期)

若**远期决定公开 / 商业化**(需重审 [[wepoker-tos-risk-boundary]]):
- **Nuitka**(Python → C → exe,看着像原生)+ **EV 代码签名证书**($200-500/年)+ **AV 白名单提交**
- 约 1 周工程量;**仅当用户解锁 ToS 红线后才执行**
